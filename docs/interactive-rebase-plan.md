## 已完成：Edit 动作

目标：计划中可选择“编辑”。Git 在该提交处暂停，用户修改工作区、暂存后继续变基。

验收：

1. 选择某条普通提交为“编辑”，执行计划。
2. 工作区进入变基中间态，提交区显示继续、跳过、中止。
3. 用户修改文件并暂存后点击继续，历史生成新的提交。
4. 合并提交不允许选择“编辑”。

## 验收记录

- `Edit`：已通过真实 Git 回归。流程为暂停 -> 暂存改动 -> 修订当前提交 -> 继续变基。
- `Autosquash`：已通过 `test-repo/codex/autosquash-lab` 验收。匹配的 `fixup!` 和
  `squash!` 自动重排到目标提交后；无法匹配的提交保留为普通提交；手动拖拽或改动作会
  覆盖自动预设。
- `计划预览`：计划表下方复用提交详情和 diff 缓存。选中提交后加载文件列表，选中文件后
  显示补丁；块/完整文件模式与行复制独立保存。预览显示当前计划顺序、动作影响、合并目标和
  前后相邻提交，拖拽或切换动作会立即反映在预览中。重写后提交会明确标记“重写后”，其子提交
  的前一提交会引用计划中的新节点，而非原始历史中的旧节点。
- `组合执行`：真实临时仓库回归已覆盖 `reword -> fixup -> pick`：改写后的提交信息与作者保留，
  fixup 内容进入该提交，后续提交继续重放。
